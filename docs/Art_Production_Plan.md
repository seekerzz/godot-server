**项目名称**: 混合现实乒乓球游戏
**版本**: 1.0

## 1. 项目概述

本文档定义了混合现实乒乓球游戏的完整美术资源生产计划，包括资源清单、制作规范、时间安排和质量标准。

**技术基础**:
- 引擎: Godot 4.6
- 渲染器: Forward Plus
- 目标平台: Windows PC
- 美术风格: 现代极简科技风

## 3. 详细资源规格

### 3.1 P0 核心资源（第一周）

#### 3.1.1 玩家球拍模型 (paddle_player)

**设计要点**:
- 抽象科技感造型，与真实手机1:1映射
- 半透明发光材质，电光蓝主题色
- 明显的"击球面"指示区域

**技术规格**:
| 参数 | 数值 |
|------|------|
| 长度 | 0.16米 |
| 宽度 | 0.075米 |
| 厚度 | 0.008米 |
| 面数限制 | < 500三角面 |
| 主色调 | 电光蓝 `#00D4FF` |
| 材质类型 | StandardMaterial3D |

**结构组成**:
1. **手机主体**: 长方体，半透明材质
2. **击球面**: 从手机底部延伸的扇形区域
3. **发光边缘**: 边缘高亮发光效果

**材质参数**:
```gdscript
albedo_color = Color(0, 0.83, 1, 0.8)
emission_enabled = true
emission = Color(0, 0.83, 1)
emission_energy_multiplier = 0.5
transparency = TRANSPARENCY_ALPHA
metallic = 0.3
roughness = 0.2
```

**输出文件**:
- `assets/models/paddle_player.glb`
- `assets/materials/mat_paddle_player.tres`

---

#### 3.1.2 AI球拍模型 (paddle_ai)

**设计要点**:
- 与玩家球拍有明显视觉区分
- 霓虹紫主题色，更强的发光效果
- 六边形核心 + 菱形拍面的几何造型

**技术规格**:
| 参数 | 数值 |
|------|------|
| 尺寸 | 同玩家球拍 (0.16×0.075×0.008米) |
| 面数限制 | < 500三角面 |
| 主色调 | 霓虹紫 `#B829DD` |
| 发光强度 | 0.7 (比玩家球拍更强) |

**视觉区分特征**:
| 特征 | 玩家球拍 | AI球拍 |
|------|---------|--------|
| 主色 | 电光蓝 | 霓虹紫 |
| 形状 | 矩形+扇形 | 六边形+菱形 |
| 发光强度 | 0.5 | 0.7 |
| 动态效果 | 静态发光 | 脉动发光 |

**输出文件**:
- `assets/models/paddle_ai.glb`
- `assets/materials/mat_paddle_ai.tres`

---

#### 3.1.3 乒乓球模型 (ball)

**设计要点**:
- 标准乒乓球尺寸
- 白色光滑材质，带轻微自发光便于追踪
- 简单高效的几何体

**技术规格**:
| 参数 | 数值 |
|------|------|
| 直径 | 0.04米 |
| 面数限制 | < 100三角面 |
| 基础颜色 | 纯白 `#FFFFFF` |
| 自发光 | 微弱白光 (emission_energy = 0.3) |

**材质参数**:
```gdscript
albedo_color = Color(1, 1, 1)
roughness = 0.1
metallic = 0.0
emission_enabled = true
emission = Color(1, 1, 1)
emission_energy_multiplier = 0.3
```

**输出文件**:
- `assets/models/ball.glb`
- `assets/materials/mat_ball.tres`

---

#### 3.1.4 球台模型 (table)

**设计要点**:
- 标准乒乓球桌尺寸
- 深炭灰桌面，电光蓝边线发光
- 金属质感边框

**技术规格**:
| 参数 | 数值 |
|------|------|
| 长度 | 2.74米 |
| 宽度 | 1.525米 |
| 高度 | 0.76米 |
| 网高 | 0.1525米 |
| 面数限制 | < 2000三角面 |

**组件分解**:
1. **桌面**: 深炭灰 `#2A2A35`，半哑光材质
2. **边线/中线**: 电光蓝 `#00D4FF`，自发光
3. **边框**: 金属质感，深灰色
4. **球网**: 半透明网格材质

**材质清单**:
- `mat_table_surface.tres` - 桌面材质
- `mat_table_line.tres` - 线条发光材质
- `mat_table_frame.tres` - 金属边框材质
- `mat_net.tres` - 球网材质

**输出文件**:
- `assets/models/table.glb`
- `assets/materials/mat_table_*.tres`

### 3.2 P1 重要资源（第二周）

#### 3.2.1 场馆环境 (environment)

**组成元素**:
1. **网格地面**: 无限延伸的发光网格
2. **雾效环境**: 深度雾营造空间感
3. **背景氛围**: 渐变暗色空间

**技术规格**:
| 元素 | 规格 |
|------|------|
| 地面网格 | 20×20米范围，1米间距 |
| 网格颜色 | 电光蓝 `#00D4FF`，透明度0.3 |
| 雾效密度 | 0.02 |
| 雾效颜色 | 深空黑 `#0A0A0F` |

**实现方式**:
- 网格：程序化生成或纹理贴图
- 雾效：Godot环境雾设置
- 背景：渐变天空盒

### 3.3 P2 增强资源（第三周）

#### 3.3.1 击球特效 (hit_effect)

**特效组成**:
1. **火花粒子**: 击球点爆发效果
2. **光晕增强**: 球拍发光脉冲
3. **冲击波**: 扩散环形效果

**粒子规格**:
| 参数 | 数值 |
|------|------|
| 粒子数量 | 20-30个 |
| 颜色 | 电光蓝到白色渐变 |
| 持续时间 | 0.3秒 |
| 扩散范围 | 0.1米半径 |

---

#### 3.3.2 轨迹特效 (trail_effect)

**设计规格**:
- 类型: 连续拖尾渲染
- 颜色渐变: 电光蓝(头部) -> 透明(尾部)
- 宽度: 0.01米，逐渐变细
- 长度: 保留最近0.5秒轨迹
- 最大点数: 50个点

### 5.1 面数预算

| 球台 | < 2000 | 1000 | 500 |
| 球拍 | < 500 | 250 | - |

### 5.2 材质规范

**StandardMaterial3D 通用设置**:
- 使用透明通道时启用 `transparency = TRANSPARENCY_ALPHA`
- 发光效果使用 `emission_enabled = true`
- 金属材质设置 `metallic` 和 `roughness`

## 6. 质量标准

### 6.1 模型质量标准

- [ ] 面数符合预算要求
- [ ] 法线方向正确
- [ ] UV展开无重叠
- [ ] 原点位置合理（通常在物体中心或底部）
- [ ] 缩放应用为1,1,1
- [ ] 无多余顶点/边/面

### 6.2 材质质量标准

- [ ] 颜色符合设计规范
- [ ] 发光效果可见但不刺眼
- [ ] 透明效果正确渲染
- [ ] 材质参数可调整
- [ ] 贴图路径正确

### 6.3 导入Godot检查清单

- [ ] 模型正确导入，无变形
- [ ] 材质正确关联
- [ ] 碰撞体设置正确
- [ ] 在场景中预览效果符合预期
- [ ] 性能开销在预算内

## 7. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 发光效果过强/过弱 | 视觉效果不佳 | 提供多档参数供程序调整 |
| 面数超预算 | 性能下降 | 制作LOD版本，优先保证P0资源 |
| 材质导入异常 | 显示错误 | 准备备用简化材质方案 |
| 文件格式兼容 | 导入失败 | 使用glb格式，测试导入流程 |
